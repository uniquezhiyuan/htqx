# 全国0.1°网格气象预报数据接口开放使用

海天气象（https://htqx.cn）现已发布全国0.1°超高分辨率网格气象预报数据集，并开放API接口，此数据集覆盖范围为中国区域，经纬度区间为15 ~ 55°N，72 ~ 136°E，区别于以往的全国市区县数据集，网格天气预报数据集具有数据量大、空间分辨率高、时间连续性强的特点，全国范围内0.1°分辨率经纬度网格形状为401×631，共有25万+个网格点，格点数量是全国市区县数据集（地点数量3191）的80倍，空间分辨率已达10公里级，时间分辨率同样为1小时，共提供未来10日240小时的超长预报时间序列，十分适合做地理信息空间分析、气象预报模型训练与检测、能源电力运输行业气候背景分析。
受数据集体积限制（日更新数据量超过40GB），此前全国0.1°分辨率网格气象预报数据难以通过网络分发应用，现在海天气象团队已从底层解决数据存储、查询检索、切片分发的核心难题，部署轻量级在线API为所有用户提供网格天气预报数据查询服务，现在用户只需要通过构建一条极简URL的方式，即可获取全国范围内任意时次任意气象要素的超高分辨率网格化数据，海量中长期的网格天气预报数据现在已经实现秒级任意调取。

## 一、数据概况

- 数据名称：全国区域未来240小时0.1°高分辨率网格天气预报数据集
- 覆盖范围：中国大陆区域（15 ~ 55°N，72 ~ 136°E）
- 预报时长：10日（240小时）
- 气象要素种类：20种（温压湿风云能天降水辐射）
- 原始数据格式：NetCDF
- 请求参数：时间、气象要素名称
- 数据源：ECMWF
- 时间分辨率：1小时
- 空间分辨率：0.1°（≤10公里）
- 更新频率：2次/天
- 更新完成时间：每日05时、17时前
- 更新方式：每日2次滚动覆盖更新
- 数据量：40GB/天
- 获取方式：API在线调取
- 使用说明：https://htqx.cn/meteo/doc/
- 示例源代码：https://htqx.cn/meteo/download/resource/HTMeteoAPI.py
- 是否支持Python SDK：暂不支持

  ## 二、数据获取方式
  
  受数据量限制不支持数据直接下载，仅支持通过API调取的方式访问数据，用户可以指定时间和气象要素名称发送GET请求获取数据，可以在浏览器端嵌入URL在线调取，也可以在应用程序中发生request请求获取数据。
  可以在海天气象官方文档查看API的详细使用说明，或下载Python示例源代码修改后直接使用调取数据。
  
  ## 三、示例URL详细说明

```python
import requests

api_key = "您的26位API Key"

time = '2026062915'  # 时间标记，固定10位长度，代表2026年4月29日15时
meteo_type = 'temperature_2m'  # 气象要素类型，地面2米气温，可选气象要素见在线文档

grid_data_url = f"https://htqx.cn/meteo/api/forecast/grid/{time}/{meteo_type}/{api_key}"
grid_data = requests.get(grid_data_url).json()
# 返回网格经纬度坐标、全国范围0.1°分辨率网格数据（401×631二维数组，覆盖范围：15~55°N，72~136°E）
print(grid_data)
```
  ![img](https://github.com/uniquezhiyuan/htqx/blob/main/figures/%E5%B1%8F%E5%B9%95%E6%88%AA%E5%9B%BE%202026-06-26%20223414.png)

以上是Python语言调取全国0.1°网格天气预报数据的示例源代码，构建request请求时需要指定2个参数，分别为**时间**和**气象要素类型**，其中`time`为时间参数，固定长度10为，代表对应日期和小时，如以上源代码中的`2026062915`为2026年6月29日15时；`meteo_type `为气象要素类型参数，可以查看官方文档了解所有支持的气象要素类型，以上源代码中的`temperature_2m`为地面2米气温，常用的气象要素类型还有`pressure_msl`、`relative_humidity_2m`、`wind_speed_10m`、`precipitation`等，分别代表海平面气压、相对湿度、风速、降水量，此外还有云量、能见度、短波辐射、直接辐射、地名散射辐射等20余种气象要素。
**注意：**
使用API调取数据时须确保气象要素名称正确，且时间参数须在当前时间未来10日的范围内，时间参数务必保持10位固定长度。

## 四、返回数据详解
![img](https://github.com/uniquezhiyuan/htqx/blob/main/figures/%E5%B1%8F%E5%B9%95%E6%88%AA%E5%9B%BE%202026-06-25%20195809.png)

上图是第三节中请求API之后返回的数据，返回数据主要分为3部分，分别是元数据信息、坐标数据和气象数据，元数据信息描述了返回数据的简要属性，包括数据对应的时间、气象要素名称、单位、经纬度边界和网格形状。坐标数据为网格点的纬度和经度坐标，包含2个一维数组，用户可以用坐标数据快速构建数据对应的经纬网。气象数据为核心内容，是用户请求的气象要素类型在对应时次的全部网格数据，覆盖范围为全国区域，为二维数组，每个数据代表该气象要素在对应坐标网格点的数值，缺失数值以-9999代替。
